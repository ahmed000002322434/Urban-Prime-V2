## Database Schema

### Firestore Collection: `listings_v2`

```typescript
{
  // Identification
  id: string;                    // Auto-generated by Firestore
  sellerId: string;              // Links to user
  ownerId: string;              // Duplicate of sellerId (for compatibility)

  // Basic Information
  title: string;                 // Listing title (5-100 chars)
  description: string;           // Full description (20-5000 chars)
  category: string;             // Category name
  brand?: string;               // Brand/manufacturer
  condition?: 'new' | 'like-new' | 'good' | 'fair' | 'poor';
  sku?: string;                 // Stock keeping unit (seller's reference)

  // Listing Type & Status
  listingType: 'sale' | 'rent' | 'both' | 'auction';
  status: 'published' | 'draft' | 'archived' | 'sold';
  isFeatured?: boolean;
  boostLevel?: string;          // Enhancement tier
  publishedAt?: Timestamp;

  // Pricing (For Sale)
  price?: number;               // Sale price
  salePrice?: number;           // Alternative sale field
  compareAtPrice?: number;      // Original price (for discounts)

  // Pricing (For Rent)
  rentalPrice?: number;         // Base rental price
  rentalPriceType?: 'daily' | 'weekly' | 'monthly';
  rentalRates?: {
    hourly?: number;
    daily?: number;
    weekly?: number;
  };
  minRentalDuration?: number;   // Minimum days
  securityDeposit?: number;     // Damage deposit
  isInstantBook?: boolean;      // Auto-approve rentals

  // Auction Fields
  auctionDetails?: {
    startingBid: number;
    currentBid: number;
    endTime: string;
    bidCount: number;
    bids: any[];
  };
  reservePrice?: number;
  buyNowPrice?: number;

  // Inventory
  stock: number;                // Available quantity
  bookedDates?: string[];       // For rental availability
  
  // Media
  images: string[];             // Image URLs
  imageUrls: string[];          // Duplicate (normalization)
  videoUrl?: string;

  // Details
  features?: string[];          // Key features list
  specifications?: Array<{      // Key-value specs
    key: string;
    value: string;
  }>;
  materials?: string[];
  careInstructions?: string[];

  // Shipping
  dimensionsIn?: {
    l: number;
    w: number;
    h: number;
  };
  weightLbs?: number;
  shippingOptions?: Array<{
    name: string;
    cost: number;
    estimatedDays: number;
  }>;
  whoPaysShipping?: 'seller' | 'buyer';

  // Additional
  owner: {                      // Owner snapshot
    id: string;
    name: string;
    avatar: string;
    businessName?: string;
  };
  avgRating: number;
  reviews: Review[];
  isVerified?: boolean;

  // System Fields
  createdAt: Timestamp;
  updatedAt: Timestamp;
  version?: string;             // Schema version
}
```

**Indexes Required:**
```
// For seller dashboard
Collection: listings_v2
Fields: (sellerId, status, createdAt)

// For category browsing
Collection: listings_v2
Fields: (status, category, createdAt)

// For search/pricing
Collection: listings_v2
Fields: (status, price, createdAt)
```

## Error Handling

All methods throw errors for failed operations:

```typescript
try {
  await listingsService.createListing(userId, data);
} catch (error) {
  // Error types:
  // - "Validation failed: field1: message1, field2: message2"
  // - "Failed to create listing: <firestore error>"
  // - "Listing not found" (on update/delete of non-existent)
  
  console.error(error.message);
  // Display to user...
}
```

**Common Errors:**
- `Validation failed` - Check all required fields and format
- `Listing not found` - Item was deleted or wrong ID
- `Failed to create/update` - Firestore permission or quota issue
- `PERMISSION_DENIED` - User lacks permission to modify listing

### Database Migration
```typescript
// Script to migrate items to new listings collection
async function migrateItems() {
  const oldItems = await getDocs(collection(db, 'items'));
  
  oldItems.forEach(async (doc) => {
    const item = doc.data();
    
    // Create in new collection with proper schema
    await addDoc(collection(db, 'listings_v2'), {
      ...item,
      sellerId: item.ownerId,
      status: 'draft',
      createdAt: item.createdAt || Timestamp.now()
    });
  });
}
```

## Support & Debugging

### Common Issues

**Q: Images not loading?**
A: Check image URLs are valid public URLs. Implement Firebase Storage upload.

**Q: Validation errors not showing?**
A: Check browser console for errors. Ensure form is validating before submit.

**Q: Listings disappear after publish?**
A: Check status filter. Unpublished listings hidden by default.

**Q: Can't create listing as non-seller?**
A: User must have seller account. Add role check to form.

**Q: Firestore quota exceeded?**
A: Implement read caching. Batch operations. Check index usage.
