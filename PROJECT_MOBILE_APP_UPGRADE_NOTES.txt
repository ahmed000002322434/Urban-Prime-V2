4) Performance and loading strategy notes

- Current frontend behavior already uses route-level code splitting (`React.lazy`) for most pages.
- New mobile shell adds lightweight UI-only interactions with minimal network overhead.
- Search overlay routes to existing browse query flow instead of adding new heavy endpoints.
- Floating mobile clutter was removed to reduce main-thread work and tap conflicts.


5) About the "10 million requests/second" target

- This cannot be guaranteed by frontend changes alone.
- To approach ultra-high throughput safely, production architecture must include:
  - Global CDN + edge caching
  - Stateless horizontally scaled API layer
  - Queue-based write buffering
  - Read replicas + partitioning strategy
  - Rate limiting + WAF + bot mitigation
  - Async processing for non-critical work
  - Full observability + autoscaling policies
- This update keeps functionality intact while reducing unnecessary client-side churn.
